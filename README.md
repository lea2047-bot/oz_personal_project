## 📖 프로젝트 소개

> 오늘 할 일, 아직도 머릿속에만 저장하시나요? ❌ 이제 더 이상 혼자 고민하며 멈춰 서지 마세요! ❌  
> 이 앱은 단순한 할 일 목록이 아닙니다. 대다수가 지향하는 '일상의 높은 진입장벽을 허물고 성취로 나아가는 첫걸음'

사용자는 **일일 할 일 목록(To-Do)**을 통해 자신의 하루를 점검하고, 진행 상황을 확인하며 동료들과 성취를 나눕니다. 
또한 공유된 목표 속에서 서로를 응원하거나 팁을 공유하며, 함께 나아가는 실행 습관을 만들어갑니다.

함께하는 즐거움을 통해 매일의 성취를 차곡차곡 쌓아가는 공간, To Do List에 오신 걸 환영합니다! 🎉

## 📑 프로젝트 규칙

### 🌿 브랜치 규칙

| 종류 | 설명 | 예시 | 비고 |
|---|---|---|---|
| main | 메인 브랜치 | main | 그대로 사용 |
| dev | 개발 브랜치 | develop | 그대로 사용 |
| feature | 기능 개발 | feature/10-signin | 로그인 |
| hotfix | 버그 수정 | hotfix-1.1.4 | |
| release | 배포 브랜치 | release-1.1 | |

> 네이밍 규칙: **타입/이슈번호-작업내용**

---

### 🌿 커밋 메세지 규칙

- **타입: 제목 (이슈번호)**
  - e.g. `feat: 메인 페이지 구현 (#5)`
  - (한 줄 띄움)
  - `- 구현 내용 1`
  - `- 구현 내용 2`

|     Type     | 설명                                                 |
| :----------: | :--------------------------------------------------- |
|   **feat**   | 새로운 기능 추가                                     |
|   **fix**    | 버그 수정                                            |
| **refactor** | 리팩토링                                             |
|  **design**  | CSS 및 사용자 UI 디자인 변경                         |
|  **style**   | 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우    |
|   **test**   | 테스트(테스트 코드 추가, 수정, 삭제)                 |
|  **chore**   | 기타 변경사항 (빌드 스크립트 수정, 패키지 매니저 등) |
|   **init**   | 프로젝트 초기 생성                                   |
|  **rename**  | 파일 혹은 폴더명 수정 또는 이동                      |
|  **remove**  | 파일을 삭제하는 작업만 수행한 경우                   |
|   **type**   | 타입 정의                                            |

### 🌿 이슈 및 PR 라벨

- **Feature** - 기능 구현
- **Bug** - 버그 관련
- **Refactor** - 코드 리팩토링
- **Setting** - 개발 환경 세팅
- **Docs** - 문서 작성 및 수정
- **Design** - 마크업 및 스타일링
- **Deploy** - 배포
- **Rename** - 파일명 수정 혹은 파일 위치 이동
